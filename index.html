What it does

Everything your
media stack needs.

Built for Sonarr and Radarr power users who want automated, intelligent searching without the complexity.

📺
Multi-instance
Run as many Sonarr and Radarr instances as you need — HD, 4K, Anime, all together.
⭐
IMDb filter
Only search content with an IMDb rating ≥ your threshold. 0 = off. Works for both series and movies.
🎯
Target resolution
Only trigger upgrades if current quality is below your target — e.g. only upgrade up to 1080p Bluray.
🎲
Random selection
Each cycle picks items randomly from your missing list.
🔔
Discord notifications
6 configurable event types — missing, upgrades, cooldown, daily limits, offline, stats.
⏱️
Intervals in minutes
Search intervals in minutes — default 30 min. Jitter prevents predictable API patterns.
🧙
First-run wizard
4-step setup wizard in the browser — no config file editing required.
🔐
Optional password
Set MEDIASTARR_PASSWORD to protect the dashboard and API. CSRF protection included.
🌐
German & English
Full UI, log messages, and Discord notifications in both languages.
Get started

Up in 60 seconds.

One command to start. The browser wizard handles the rest.

1
Pull and run

One Docker Compose line — port 7979, a data volume, done.

2
Open the browser wizard

http://your-server:7979 — The setup wizard opens automatically.

3
Add your instances

Enter URLs and API keys, test the connection, done.

docker-compose.yml
services:
  mediastarr:
    image: kroeberd/mediastarr:latest
    container_name: mediastarr
    restart: unless-stopped
    ports:
      - "7979:7979"
    volumes:
      - /appdata/mediastarr:/data
    # environment:
    # - MEDIASTARR_PASSWORD=change-me

# Then open http://your-server:7979

Why Mediastarr

Built right.
From scratch.

In February 2026, the most popular *arr automation tool was shut down overnight after a public security audit exposed critical vulnerabilities. Mediastarr was built with security as a foundation — not an afterthought.

🔓
What happened — February 2026
A public security audit of Huntarr v9.4.2 uncovered 21 vulnerabilities — 7 critical, 6 high. The most severe: a single unauthenticated HTTP request to POST /api/settings/general returned every API key for every connected *arr app in cleartext — Sonarr, Radarr, Prowlarr, Lidarr — with no login required. Additional findings included full account takeover via unauthenticated 2FA enrollment and the ability to rewrite the entire configuration without any session. Shortly after the report was posted to Reddit, the Huntarr GitHub repository, subreddit and Discord were taken offline. The project is no longer maintained.
⚠ What about the forks?
Several community forks appeared after the shutdown. They fix specific findings from the audit — but they inherit the same codebase and the same development process. As the original security researcher noted: "Fixing 21 specific findings doesn\'t fix the process that created them. Without code review, without basic automated testing, without anyone who understands security fundamentals actually reviewing what ships — the next batch of features will introduce the next batch of vulnerabilities." Mediastarr was built independently, from scratch, before and after these events — with security practices embedded from day one.
