What it does

Everything your
media stack needs.

Built for Sonarr and Radarr power users who want automated, intelligent searching without the complexity.

📺
Multi-instance support
Run as many Sonarr and Radarr instances as you need — HD, 4K, Anime, all together. Each gets its own named card with live stats.
🎲
Random selection
Each hunt cycle picks items randomly from your missing list — so all content gets coverage, not just whatever's at the top.
⏳
Cooldown & daily limits
Configurable per-item cooldown (1–365 days) and a global daily search cap. SQLite history tracks every search with timestamps and retry counts.
🔔
Discord notifications
6 configurable event types — missing searches, upgrades, cooldown expiry, daily limits, offline instances, and periodic stats reports.
🎲
Jitter scheduling
Add random delay to hunt intervals to avoid predictable API patterns. Minimum 15-minute interval is always enforced regardless of settings.
🌐
German & English
Full UI, log messages, and Discord notifications in both languages. Switch instantly from the sidebar — no restart required.
🧙
First-run wizard
4-step setup wizard in the browser — no config file editing, no environment variables required. Add instances, test connections, configure Discord, done.
📊
Statistics dashboard
Searches per instance, by release year, search types breakdown, result distribution donut, and 24h activity timeline — all live.
🎨
3 themes
Dark, Light, and OLED Black. Stored server-side so it persists across devices and browser sessions.
Get started

Up in 60 seconds.

One command to start. The browser wizard handles the rest.

1
Pull and run

One Docker Compose line — port 7979, a data volume, done.

2
Open the browser wizard

Navigate to http://your-server:7979 — the setup wizard opens automatically.

3
Add your instances

Enter your Sonarr and Radarr URLs and API keys. Test the connection, click finish — hunting starts automatically after the first interval.

docker-compose.yml
services:
  mediastarr:
    image: kroeberd/mediastarr:latest
    container_name: mediastarr
    restart: unless-stopped
    ports:
      - "7979:7979"
    volumes:
      - /appdata/mediastarr:/data

# Then open http://your-server:7979
Notifications

Stay in the loop.

Discord webhook with 6 configurable events and built-in rate-limit protection — no more 429 errors.

Discord Webhook Integration
Paste your webhook URL in Settings → Discord. Each event type can be toggled individually. Configurable rate-limit cooldown (default 5s) prevents Discord 429 errors.
